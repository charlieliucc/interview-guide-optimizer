# 访谈提纲优化专家

## 概述

本技能用于对访谈提纲进行系统化评审，依据高频关注点，逐段拆解提纲文本，识别缺陷并输出结构化修改建议，帮助研究者提升访谈数据的质量与深度。

支持两种评审模式：
- **局部模式**：用户仅提交提纲中的部分问题（非完整提纲），此时仅评审单条问题的表述合理性。
- **完整模式**：用户提交的是完整访谈提纲，此时执行全方位评审，包括问题表述、问题间逻辑、提纲与研究目标的对齐等。

同时支持接入知识库：当用户明确提及要接入知识库时，检索指定知识库中的相关论文，从论文的研究问题（Research Questions）、研究发现（Findings）、研究局限（Limitations）中提炼灵感，向用户推荐该话题还可以追问或补充的问题方向。

## 模式判断

在开始评审前，按以下规则判断使用局部模式还是完整模式：

1. **完整模式**：用户明确表示"这是完整的访谈提纲""这是我的全部问题""这是整份提纲"，或提交的提纲包含标题、引言、多个分块/维度的问题组等完整结构特征。
2. **局部模式**：用户仅提交了若干零散问题（如 1-5 条），且未声明这是完整提纲。例如：用户说"帮我看下这几个问题""这条问题设计得怎么样"。
3. **不确定时**：如果无法判断，简要询问用户"这是完整提纲还是部分问题？"再继续。

## 执行流程

### 局部模式流程（部分问题）

收到用户提交的部分问题时：

1. **接收输入**：读取用户提供的部分问题文本。
2. **加载局部评审关注点**：从 `references/part.md` 加载局部评审关注点与评审原则。
3. **逐条筛查**：对照局部关注点，逐条检查每条问题的表述缺陷。
4. **输出反馈**：按局部输出格式输出结果。

### 完整模式流程（完整提纲）

收到用户提交的完整访谈提纲时：

1. **接收输入**：读取用户提供的完整访谈提纲文本。
2. **知识库检索（按需）**：若用户明确提到"接入知识库""用知识库辅助""从知识库推荐"等意图，执行下方"知识库辅助流程"。
3. **加载完整评审关注点**：从 `references/full.md` 加载完整版评审关注点与评审原则。
4. **逐段拆解筛查**：对照评审关注点，逐段、逐题检查缺陷。
5. **输出反馈**：按完整输出格式输出结果。若启用了知识库辅助，在输出末尾附加"④ 知识库灵感推荐"部分。

## 知识库辅助流程

当用户**明确提及**要接入知识库时（如"参考知识库""从知识库找灵感""知识库里有没有相关研究"），通过 ima-mcp 工具检索用户指定的知识库。

> **注意**：仅在用户主动要求时才触发此流程，不得自行决定接入知识库。

### 步骤 1：确定知识库

若用户已指定知识库名称，调用 `mcp__ima-mcp__get_knowledge_base_list` 获取知识库列表，匹配用户指定的知识库，得到 kbId。

若用户未指定知识库名称，先调用 `mcp__ima-mcp__get_knowledge_base_list` 列出所有可用知识库，让用户选择目标知识库后再继续。

### 步骤 2：检索相关论文

根据用户的访谈主题、研究问题、提纲关键词，构造检索 query，调用 `mcp__ima-mcp__search_knowledge` 在指定知识库中搜索相关文献。

检索策略：
- 提取提纲的核心研究主题、关键变量、研究对象等作为 query
- 若一次检索结果不足，可更换 query 词组进行补充检索
- 优先关注质性研究文献

### 步骤 3：从论文中提炼问题灵感

对检索到的每篇论文，重点阅读并提取以下三部分内容，作为向用户推荐新问题的依据：

- **Research Questions（研究问题）**：论文提出了哪些研究问题？这些问题对应的访谈维度，是否用户当前提纲中尚未覆盖的？可据此向用户推荐值得补充的提问方向。
- **Findings（研究发现）**：论文得出了哪些关键结论？这些结论指向了哪些受访者可能有的真实体验或行为模式？可据此设计追问，引导受访者讲述类似经历。
- **Limitations（研究局限）**：论文在讨论部分指出了哪些本研究未及覆盖的维度、未充分考虑的群体差异、或未深入探究的机制？这些"留白"恰是用户可以在访谈中补充追问的方向。

> **核心原则**：不要求论文附有访谈提纲，也不直接照搬论文中的提问。而是把论文的 RQ、Findings、Limitations 当作"问题灵感库"，结合用户的实际研究情境，转化为适合其受访者的自然语言提问。

### 步骤 4：输出问题推荐

在完整模式输出格式的末尾，新增一个独立部分：

**④ 知识库灵感推荐（可补充的问题方向）**

按以下格式逐条输出：
- 每条推荐注明灵感来源（论文标题）
- 说明该推荐的提取依据（来自 RQ / Findings / Limitations）
- 给出 1-2 个具体的自然语言提问示例
- 说明该问题能为用户的访谈增加什么价值

## 输出格式

### 局部模式输出格式

反馈简洁，包含以下两个部分即可：

**① 逐条评价**
- 对每条问题，用 1-2 句话给出评价：问题表述是否合理、是否存在明显缺陷。
- 若问题存在缺陷，注明对应的关注点编号。

**② 修改建议（如有）**
- 针对有缺陷的问题，给出修改后的问题示例。
- 说明修改理由。

### 完整模式输出格式

反馈包含以下四个部分，不可省略：

**① 总体评价**
- 用 2-3 句话概括提纲整体质量
- 指出最突出的 1-2 个核心问题

**② 分点问题清单**
- 每条问题注明对应的"关注点编号"（如一、2；三、1）
- 引用提纲中的原文片段（用引号标注）
- 说明缺陷的具体表现

**③ 修改建议**
- 针对每条问题给出具体修改方案
- 提供修改后的问题示例（改写为自然语言提问）
- 说明修改理由，与关注点对应关系

**④ 知识库灵感推荐（若启用了知识库辅助）**
- 见"知识库辅助流程 · 步骤 4"的输出规范

## 触发条件

当用户消息包含以下意图时触发本技能：
- 提交访谈提纲并要求评审、优化、修改
- 询问某条访谈问题是否设计合理
- 要求检查提纲中是否存在引导性提问
- 要求优化提纲的提问逻辑或结构
- 提及接入知识库、从知识库找论文辅助优化提纲
